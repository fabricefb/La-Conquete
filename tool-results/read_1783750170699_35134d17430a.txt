     3вҶ’     3вҶ’## Color Philosophy
     4вҶ’     4вҶ’
     5вҶ’     5вҶ’GLM uses a **mood-driven dynamic color system** instead of fixed named palettes. Colors are constructed from three dimensions:
     6вҶ’     6вҶ’
     7вҶ’     7вҶ’### Three Dimensions of Document Color
     8вҶ’     8вҶ’
     9вҶ’     9вҶ’| Dimension | Description | Range |
    10вҶ’    10вҶ’|-----------|-------------|-------|
    11вҶ’    11вҶ’| **Temperature** | Warm вҶ” Cool | Warm (consulting, education) вҶ” Cool (tech, medical) |
    12вҶ’    12вҶ’| **Weight** | Light вҶ” Heavy | Light (resume, proposal) вҶ” Heavy (legal, academic) |
    13вҶ’    13вҶ’| **Energy** | Calm вҶ” Active | Calm (official, contract) вҶ” Active (report, presentation) |
    14вҶ’    14вҶ’
    15вҶ’    15вҶ’### Color Token System
    16вҶ’    16вҶ’
    17вҶ’    17вҶ’Every document uses 5 color tokens. These are **computed** based on the document's mood, not selected from a fixed list.
    18вҶ’    18вҶ’
    19вҶ’    19вҶ’| Token | Role | Guidance |
    20вҶ’    20вҶ’|-------|------|----------|
    21вҶ’    21вҶ’| `primary` | Headings, cover title | Dark, authoritative. Derived from Temperature + Weight. |
    22вҶ’    22вҶ’| `body` | Body text | Near-black with subtle warmth/coolness. Always high contrast. |
    23вҶ’    23вҶ’| `secondary` | Captions, footnotes | Mid-tone gray. Legible but visually recessive. |
    24вҶ’    24вҶ’| `accent` | Table headers, lines, links | The "personality" color. Reflects document Energy. |
    25вҶ’    25вҶ’| `surface` | Table alternating rows, card backgrounds | Very light tint of accent or neutral. |
    26вҶ’    26вҶ’

   157вҶ’   157вҶ’### Visual Profile Color Guidance
   158вҶ’   158вҶ’
   159вҶ’   159вҶ’For **Profile B (Visual)** scenes вҖ” resume, copywriting, and other non-formal documents вҖ” prefer **warm neutral** tones aligned with the "Invisible Precision" design philosophy:
   160вҶ’   160вҶ’
   161вҶ’   161вҶ’- **Body text**: Use warm dark neutrals (`#37352F`, `#303030`, `#3A3430`) instead of cool blue-grays. This reduces eye strain.
   162вҶ’   162вҶ’- **Surface/background**: Use warm near-white (`#F7F7F7`, `#FAFAF8`, `#FBF9F7`) instead of cool tints.
   163вҶ’   163вҶ’- **Accent colors**: Use sparingly and only for functional differentiation (section headers, key metrics, links). 95% of the document should remain monochromatic (black, white, gray).
   164вҶ’   164вҶ’- **Tables**: Prefer the **Zebra Stripe** style (see Table Styles В§2) вҖ” hierarchy through background contrast with minimal borders. Fallback: Horizontal-Only.
   165вҶ’   165вҶ’
   166вҶ’   166вҶ’This does NOT apply to Profile A (Formal) scenes (report, academic, contract, official-doc, exam), which must retain pure black `"000000"` body text per regulatory standards.
   167вҶ’   167вҶ’
   168вҶ’   168вҶ’### Custom Color Generation
   169вҶ’   169вҶ’
   170вҶ’   170вҶ’When the pre-defined recipes don't fit, construct colors using these rules:
   171вҶ’   171вҶ’
   172вҶ’   172вҶ’1. **primary**: Start from `hsl(hue, 25-40%, 10-18%)` вҖ” dark, desaturated
   173вҶ’   173вҶ’2. **body**: primary lightened 5-8% вҖ” readable dark
   174вҶ’   174вҶ’3. **secondary**: primary lightened 30-40% вҖ” clearly subordinate
   175вҶ’   175вҶ’4. **accent**: Choose a hue reflecting the domain, `hsl(domainHue, 30-50%, 45-55%)`
   176вҶ’   176вҶ’5. **surface**: accent desaturated to 5-10%, lightened to 96-98%
   177вҶ’   177вҶ’
   178вҶ’   178вҶ’**Contrast check**: body text on white must achieve WCAG AA (вүҘ4.5:1). All recipes above pass this.

   255вҶ’   255вҶ’## Table Styles
   256вҶ’   256вҶ’
   257вҶ’   257вҶ’**Profile routing:** Profile A (Formal: report, academic, contract, exam) вҶ’ Three-Line Table or Horizontal-Only Table. Profile B (Visual: resume, copywriting) вҶ’ Zebra Stripe (preferred) or Horizontal-Only Table.
   258вҶ’   258вҶ’
   259вҶ’   259вҶ’### 1. Three-Line Table (дёүзәҝиЎЁ) вҖ” Academic
   260вҶ’   260вҶ’
   261вҶ’   261вҶ’Only three horizontal lines: top of table, bottom of header, bottom of table.
   262вҶ’   262вҶ’
   263вҶ’   263вҶ’```js
   264вҶ’   264вҶ’const threeLineTable = new Table({
   265вҶ’   265вҶ’  width: { size: 100, type: WidthType.PERCENTAGE },
   266вҶ’   266вҶ’  borders: {
   267вҶ’   267вҶ’    top: { style: BorderStyle.SINGLE, size: 4, color: "000000" },
   268вҶ’   268вҶ’    bottom: { style: BorderStyle.SINGLE, size: 4, color: "000000" },
   269вҶ’   269вҶ’    left: { style: BorderStyle.NONE },
   270вҶ’   270вҶ’    right: { style: BorderStyle.NONE },
   271вҶ’   271вҶ’    insideHorizontal: { style: BorderStyle.NONE },
   272вҶ’   272вҶ’    insideVertical: { style: BorderStyle.NONE },
   273вҶ’   273вҶ’  },
   274вҶ’   274вҶ’  rows: [
   275вҶ’   275вҶ’    new TableRow({
   276вҶ’   276вҶ’      children: headerCells.map(text => new TableCell({
   277вҶ’   277вҶ’        children: [new Paragraph({ children: [new TextRun({ text, bold: true, size: 21 })] })],
   278вҶ’   278вҶ’        borders: {
   279вҶ’   279вҶ’          bottom: { style: BorderStyle.SINGLE, size: 2, color: "000000" },
   280вҶ’   280вҶ’          top: { style: BorderStyle.NONE },
   281вҶ’   281вҶ’          left: { style: BorderStyle.NONE },
   282вҶ’   282вҶ’          right: { style: BorderStyle.NONE },
   283вҶ’   283вҶ’        },
   284вҶ’   284вҶ’        margins: { top: 60, bottom: 60, left: 120, right: 120 },
   285вҶ’   285вҶ’      })),
   286вҶ’   286вҶ’    }),
   287вҶ’   287вҶ’    // ... data rows with all borders NONE
   288вҶ’   288вҶ’  ],
   289вҶ’   289вҶ’});
   290вҶ’   290вҶ’```

   332вҶ’   332вҶ’|-------|---------|-------------|
   333вҶ’   333вҶ’| `table.headerBg` | Zebra Stripe, Horizontal-Only | Table header background color |
   334вҶ’   334вҶ’| `table.headerText` | Zebra Stripe, Horizontal-Only | Table header text color (must pass WCAG AA contrast) |
   335вҶ’   335вҶ’| `table.accentLine` | Three-Line | Top/bottom/header-bottom line color |
   336вҶ’   336вҶ’| `table.innerLine` | Horizontal-Only | Inner horizontal separator line color |

   339вҶ’   339вҶ’**Usage:**
   340вҶ’   340вҶ’```js

   344вҶ’   344вҶ’// Zebra: use t.headerBg, t.headerText, t.surface
   345вҶ’   345вҶ’// Horizontal: use t.headerBg, t.headerText, t.innerLine
   346вҶ’   346вҶ’```

   348вҶ’   348вҶ’**вҡ пёҸ High-saturation accent override**: For DM-1, FG-1, and SN-2, the table colors are intentionally darkened/desaturated relative to the cover accent. Bright accent colors that look good on dark cover backgrounds are too eye-straining on white body pages. Always use `palette.table.*` for tables, never the raw `palette.accent`.

   353вҶ’   353вҶ’## Cover Page Design System
   354вҶ’   354вҶ’
   355вҶ’   355вҶ’### Design Philosophy
   356вҶ’   356вҶ’
   357вҶ’   357вҶ’Covers use **7 validated layout recipes** + **parameterized variants** instead of free combination. Each recipe's background, layout, and decoration are visually verified. Differentiation comes from **palette Г— font size Г— content variation**.
   358вҶ’   358вҶ’
   359вҶ’   359вҶ’**Architecture principle:** All recipes use a **single 16838 outer wrapper table** (one row, exact height). Recipes R1/R2/R3 use **ZERO nested tables** вҖ” all decoration is achieved via **paragraph borders** (left/right/top/bottom). This ensures maximum cross-engine stability (MS Office + WPS).
   360вҶ’   360вҶ’
   361вҶ’   361вҶ’### Cover Color Palettes (Dark Mode + Light Mode)
   362вҶ’   362вҶ’
   363вҶ’   363вҶ’Each palette defines 3 core colors (Background, Primary, Accent) + derived cover/table tokens.
   364вҶ’   364вҶ’

   566вҶ’   566вҶ’function applyLongTitleOverride(result, titleLength) {
   567вҶ’   567вҶ’  if (!result || !result.recipe) return result;

   570вҶ’   570вҶ’  // R6 (editorial) is designed for short titles only (вүӨ20 chars, single line)
   571вҶ’   571вҶ’  // Long titles вҶ’ fall back to R1 (handles long titles best)

   573вҶ’   573вҶ’    return { recipe: "R1", palette: "WM-1" }; // ED-1 has no dark bg, use WM-1 (warm teal)

   576вҶ’   576вҶ’  if (titleLength > 20 && ["R3", "R4"].includes(result.recipe)) {
   577вҶ’   577вҶ’    return { recipe: "R1", palette: result.palette };
   578вҶ’   578вҶ’  }

   647вҶ’   647вҶ’### вҡ пёҸ Cover Content Overflow Prevention (Mandatory)
   648вҶ’   648вҶ’
   649вҶ’   649вҶ’1. Cover section page margin is 0; total content height вүӨ 15638 twips (1200 twips safety margin for cross-engine compatibility вҖ” MS Office renders large fonts taller than calculated).
   650вҶ’   650вҶ’2. Color block Table `height` must use `rule: "exact"` (never `"atLeast"`).
   651вҶ’   651вҶ’3. Each recipe code includes height budget annotations вҖ” verify during generation.
   652вҶ’   652вҶ’4. **`verticalAlign` must always be `"top"`**. Never use `"center"` or `"bottom"` in exact-height rows вҖ” content will be clipped or overflow. Use `spacing.before` on the first paragraph for vertical positioning.
   653вҶ’   653вҶ’5. **Title font size MUST be dynamically calculated via `calcTitleLayout()`** (see below). Never hardcode font sizes above 40pt for cover titles. Every recipe MUST call `calcTitleLayout()` before building title paragraphs.
   654вҶ’   654вҶ’6. **Never use `margins.top`/`margins.bottom` for vertical positioning inside exact-height cells**: Cell margins reduce available height unpredictably across MS Office and WPS. Use `spacing.before` on the first paragraph instead. Only `margins.left`/`margins.right` are safe.
   655вҶ’   655вҶ’7. **Dynamic spacing is mandatory**: Use `calcCoverSpacing()` to compute `spacing.before` values dynamically based on content element count and title line count. Never use fixed large spacing values (e.g., `before: 4500`) that assume a specific title length.
   656вҶ’   656вҶ’8. **Cover must be a single-page section**: The cover section must produce exactly ONE page. If content overflows to a second page, it means the height budget is violated. Common overflow causes and fixes:
   657вҶ’   657вҶ’   - Title font too large вҶ’ use `calcTitleLayout()` to auto-reduce
   658вҶ’   658вҶ’   - Too many meta lines вҶ’ reduce font size or remove less important lines
   659вҶ’   659вҶ’   - Fixed `spacing.before` values too large вҶ’ use `calcCoverSpacing()` for dynamic values
   660вҶ’   660вҶ’   - Subtitle + English label + meta lines combined exceed budget вҶ’ calculate total and reduce spacing
   661вҶ’   661вҶ’9. **Cover wrapper table MUST use explicit `allNoBorders`**: The outer 16838 wrapper table and ALL nested tables inside the cover MUST set borders to NONE explicitly. Never rely on docx-js default borders (`single/auto/sz=4`). Default borders add ~8 twips per edge, which causes MS Office to calculate a total height slightly exceeding 16838 вҶ’ content overflows to a blank page 2. WPS is more lenient but MS Office is strict. **This is the #1 cause of "blank page 2 in MS Office but not in WPS".**
   662вҶ’   662вҶ’
   663вҶ’   663вҶ’```js
   664вҶ’   664вҶ’// вң… MANDATORY: Define and use allNoBorders for every cover table
   665вҶ’   665вҶ’const NB = { style: BorderStyle.NONE, size: 0, color: "FFFFFF" };

   672вҶ’   672вҶ’  // ...

   674вҶ’   674вҶ’```
   675вҶ’   675вҶ’

   712вҶ’   712вҶ’   }

   718вҶ’   718вҶ’### вҡ пёҸ Cover Title Layout вҖ” calcTitleLayout() (Mandatory for ALL Recipes)
   719вҶ’   719вҶ’
   720вҶ’   720вҶ’**Every cover recipe MUST use `calcTitleLayout()` to determine title font size and line breaks.** Hardcoding font sizes or passing the full title as a single TextRun is FORBIDDEN.
   721вҶ’   721вҶ’
   722вҶ’   722вҶ’**Every paragraph with font size > body text MUST set explicit line spacing** to prevent top clipping:
   723вҶ’   723вҶ’```js
   724вҶ’   724вҶ’// вҳ… MANDATORY: prevent inherited small line spacing from clipping large fonts
   725вҶ’   725вҶ’spacing: { line: Math.ceil(titlePt * 23), lineRule: "atLeast", after: 100 }
   726вҶ’   726вҶ’// Example: 36pt вҶ’ line: 828; 44pt вҶ’ line: 1012
   727вҶ’   727вҶ’```
   728вҶ’   728вҶ’Without this, the paragraph inherits body text line spacing (e.g., 560tw), which is shorter than the font height вҶ’ top of characters gets clipped.
   729вҶ’   729вҶ’
   730вҶ’   730вҶ’```js
   731вҶ’   731вҶ’/**
   732вҶ’   732вҶ’ * Calculate safe font size and smart line breaks for cover titles.
   733вҶ’   733вҶ’ * MUST be called by every recipe before building title paragraphs.
   734вҶ’   734вҶ’ *
   735вҶ’   735вҶ’ * @param {string}  title          Full title string
   736вҶ’   736вҶ’ * @param {number}  maxWidthTwips  Available width for title text (twips, after subtracting margins/padding)
   737вҶ’   737вҶ’ * @param {number}  preferredPt    Desired max font size in pt (default 40)
   738вҶ’   738вҶ’ * @param {number}  minPt          Minimum allowed font size in pt (default 24)
   739вҶ’   739вҶ’ * @returns {{ titlePt: number, titleLines: string[] }}
   740вҶ’   740вҶ’ */
   741вҶ’   741вҶ’function calcTitleLayout(title, maxWidthTwips, preferredPt = 40, minPt = 24) {
   742вҶ’   742вҶ’  // Each CJK character width вүҲ pt Г— 20 twips
   743вҶ’   743вҶ’  const charWidth = (pt) => pt * 20;
   744вҶ’   744вҶ’  const charsPerLine = (pt) => Math.floor(maxWidthTwips / charWidth(pt));
   745вҶ’   745вҶ’
   746вҶ’   746вҶ’  // Try from preferredPt downward until title fits in вүӨ 3 lines
   747вҶ’   747вҶ’  let titlePt = preferredPt;
   748вҶ’   748вҶ’  let lines;
   749вҶ’   749вҶ’  while (titlePt >= minPt) {
   750вҶ’   750вҶ’    const cpl = charsPerLine(titlePt);
   751вҶ’   751вҶ’    if (cpl < 2) { titlePt -= 2; continue; }
   752вҶ’   752вҶ’    lines = splitTitleLines(title, cpl);
   753вҶ’   753вҶ’    if (lines.length <= 3) break;
   754вҶ’   754вҶ’    titlePt -= 2;
   755вҶ’   755вҶ’  }
   756вҶ’   756вҶ’
   757вҶ’   757вҶ’  // If still > 3 lines at minPt, force 3 lines
   758вҶ’   758вҶ’  if (!lines || lines.length > 3) {
   759вҶ’   759вҶ’    const cpl = charsPerLine(minPt);
   760вҶ’   760вҶ’    lines = splitTitleLines(title, cpl);
   761вҶ’   761вҶ’    titlePt = minPt;
   762вҶ’   762вҶ’  }
   763вҶ’   763вҶ’
   764вҶ’   764вҶ’  return { titlePt, titleLines: lines };
   765вҶ’   765вҶ’}
   766вҶ’   766вҶ’
   767вҶ’   767вҶ’/**
   768вҶ’   768вҶ’ * Smart Chinese title line-breaking вҖ” breaks at semantic boundaries, never mid-word.
   769вҶ’   769вҶ’ *
   770вҶ’   770вҶ’ * Rules:
   771вҶ’   771вҶ’ * 1. Prefer breaking after particles, punctuation, connectors, underscores, spaces
   772вҶ’   772вҶ’ * 2. Never split a compound word (e.g., "з®ЎзҗҶи§„иҢғ" must not become "з®ЎзҗҶи§„" + "иҢғ")
   773вҶ’   773вҶ’ * 3. No single-character orphan on the last line вҖ” merge into previous line
   774вҶ’   774вҶ’ * 4. If no good break point found within 60-130% of charsPerLine, break at charsPerLine
   775вҶ’   775вҶ’ *
   776вҶ’   776вҶ’ * @param {string} title        Full title string
   777вҶ’   777вҶ’ * @param {number} charsPerLine Max characters per line at current font size
   778вҶ’   778вҶ’ * @returns {string[]}          Array of line strings
   779вҶ’   779вҶ’ */
   780вҶ’   780вҶ’function splitTitleLines(title, charsPerLine) {
   781вҶ’   781вҶ’  if (title.length <= charsPerLine) return [title];
   782вҶ’   782вҶ’
   783вҶ’   783вҶ’  // Characters that are safe break points (break AFTER these)
   784вҶ’   784вҶ’  const breakAfter = new Set([
   785вҶ’   785вҶ’    ...'пјҢгҖӮгҖҒпјӣпјҡпјҒпјҹ',              // CJK punctuation
   786вҶ’   786вҶ’    ...'зҡ„дёҺе’ҢеҸҠд№ӢеңЁдәҺдёә',            // CJK particles/prepositions
   787вҶ’   787вҶ’    ...'-_вҖ”вҖ“В·/',                     // connectors
   788вҶ’   788вҶ’    ...' \t',                         // whitespace
   789вҶ’   789вҶ’  ]);
   790вҶ’   790вҶ’
   791вҶ’   791вҶ’  const lines = [];
   792вҶ’   792вҶ’  let remaining = title;
   793вҶ’   793вҶ’
   794вҶ’   794вҶ’  while (remaining.length > charsPerLine) {
   795вҶ’   795вҶ’    let breakAt = -1;
   796вҶ’   796вҶ’
   797вҶ’   797вҶ’    // Search backward from charsPerLine to 60% for a break point
   798вҶ’   798вҶ’    for (let i = charsPerLine; i >= Math.floor(charsPerLine * 0.6); i--) {
   799вҶ’   799вҶ’      if (i < remaining.length && breakAfter.has(remaining[i - 1])) {
   800вҶ’   800вҶ’        breakAt = i;
   801вҶ’   801вҶ’        break;
   802вҶ’   802вҶ’      }
   803вҶ’   803вҶ’    }
   804вҶ’   804вҶ’
   805вҶ’   805вҶ’    // If not found, search forward up to 130%
   806вҶ’   806вҶ’    if (breakAt === -1) {
   807вҶ’   807вҶ’      const limit = Math.min(remaining.length, Math.ceil(charsPerLine * 1.3));
   808вҶ’   808вҶ’      for (let i = charsPerLine + 1; i < limit; i++) {
   809вҶ’   809вҶ’        if (breakAfter.has(remaining[i - 1])) {
   810вҶ’   810вҶ’          breakAt = i;
   811вҶ’   811вҶ’          break;
   812вҶ’   812вҶ’        }
   813вҶ’   813вҶ’      }
   814вҶ’   814вҶ’    }
   815вҶ’   815вҶ’
   816вҶ’   816вҶ’    // Last resort: break at charsPerLine, but avoid splitting compound CJK words
   817вҶ’   817вҶ’    if (breakAt === -1) {
   818вҶ’   818вҶ’      breakAt = charsPerLine;
   819вҶ’   819вҶ’      // If both chars at the break boundary are CJK (likely a compound word),
   820вҶ’   820вҶ’      // step back 1 char to keep the word together
   821вҶ’   821вҶ’      const prevChar = remaining[breakAt - 1];
   822вҶ’   822вҶ’      const nextChar = remaining[breakAt];
   823вҶ’   823вҶ’      if (prevChar && nextChar &&
   824вҶ’   824вҶ’          !breakAfter.has(prevChar) && !breakAfter.has(nextChar) &&
   825вҶ’   825вҶ’          /[\u4e00-\u9fff]/.test(prevChar) && /[\u4e00-\u9fff]/.test(nextChar)) {
   826вҶ’   826вҶ’        breakAt = breakAt - 1;
   827вҶ’   827вҶ’      }
   828вҶ’   828вҶ’    }
   829вҶ’   829вҶ’
   830вҶ’   830вҶ’    lines.push(remaining.slice(0, breakAt).trim());
   831вҶ’   831вҶ’    remaining = remaining.slice(breakAt).trim();
   832вҶ’   832вҶ’  }
   833вҶ’   833вҶ’  if (remaining) lines.push(remaining);
   834вҶ’   834вҶ’
   835вҶ’   835вҶ’  // Prevent single-character orphan on last line вҖ” merge into previous
   836вҶ’   836вҶ’  if (lines.length > 1 && lines[lines.length - 1].length <= 2) {
   837вҶ’   837вҶ’    const last = lines.pop();
   838вҶ’   838вҶ’    lines[lines.length - 1] += last;
   839вҶ’   839вҶ’  }
   840вҶ’   840вҶ’
   841вҶ’   841вҶ’  return lines;
   842вҶ’   842вҶ’}
   843вҶ’   843вҶ’
   844вҶ’   844вҶ’/**
   845вҶ’   845вҶ’ * Calculate dynamic spacing values for cover elements to fit within page height.
   846вҶ’   846вҶ’ *
   847вҶ’   847вҶ’ * @param {object} params
   848вҶ’   848вҶ’ * @param {number} params.titleLineCount   Number of title lines
   849вҶ’   849вҶ’ * @param {number} params.titlePt          Title font size in pt
   850вҶ’   850вҶ’ * @param {boolean} params.hasSubtitle     Whether subtitle exists
   851вҶ’   851вҶ’ * @param {boolean} params.hasEnglishLabel Whether English label exists
   852вҶ’   852вҶ’ * @param {number} params.metaLineCount    Number of meta info lines
   853вҶ’   853вҶ’ * @param {number} params.fixedHeight      Sum of fixed-height elements (color strips, accent bars, footer) in twips
   854вҶ’   854вҶ’ * @param {number} params.pageHeight       Total page height in twips (default 16838)
   855вҶ’   855вҶ’ * @returns {{ topSpacing, midSpacing, bottomSpacing }}  Spacing values in twips
   856вҶ’   856вҶ’ */
   857вҶ’   857вҶ’function calcCoverSpacing(params) {
   858вҶ’   858вҶ’  const {
   859вҶ’   859вҶ’    titleLineCount = 1, titlePt = 36, hasSubtitle = false,
   860вҶ’   860вҶ’    hasEnglishLabel = false, metaLineCount = 0,
   861вҶ’   861вҶ’    fixedHeight = 800, pageHeight = 16838,
   862вҶ’   862вҶ’    marginTop = 0, marginBottom = 0,   // вҳ… NEW: pass actual section margins
   863вҶ’   863вҶ’  } = params;
   864вҶ’   864вҶ’
   865вҶ’   865вҶ’  // вҳ… Safety margin: 1200 twips (cross-engine: MS Office renders large fonts
   866вҶ’   866вҶ’  // taller than calculated; extra 400tw buffer prevents footer clipping)
   867вҶ’   867вҶ’  const SAFETY = 1200;
   868вҶ’   868вҶ’  // вҳ… Subtract page margins from available height (cover section may have margins)
   869вҶ’   869вҶ’  const usableHeight = pageHeight - marginTop - marginBottom - SAFETY;
   870вҶ’   870вҶ’
   871вҶ’   871вҶ’  // вҳ… Accurate height estimation per element:
   872вҶ’   872вҶ’  const titleHeight = titleLineCount * (titlePt * 23 + 200);
   873вҶ’   873вҶ’  const subtitleHeight = hasSubtitle ? (12 * 23 + 600) : 0;
   874вҶ’   874вҶ’  const englishLabelHeight = hasEnglishLabel ? (9 * 23 + 600) : 0;
   875вҶ’   875вҶ’  const metaHeight = metaLineCount * (10 * 23 + 100);
   876вҶ’   876вҶ’
   877вҶ’   877вҶ’  // вҳ… Account for implicit paragraph heights:
   878вҶ’   878вҶ’  const implicitParaHeight = 3 * 300;
   879вҶ’   879вҶ’
   880вҶ’   880вҶ’  const contentHeight = titleHeight + subtitleHeight + englishLabelHeight +
   881вҶ’   881вҶ’                        metaHeight + fixedHeight + implicitParaHeight;
   882вҶ’   882вҶ’
   883вҶ’   883вҶ’  const remainingSpace = usableHeight - contentHeight;
   884вҶ’   884вҶ’  const safeRemaining = Math.max(remainingSpace, 400);
   885вҶ’   885вҶ’
   886вҶ’   886вҶ’  // вҳ… Footer protection: bottomSpacing must be вүҘ FOOTER_MIN to prevent
   887вҶ’   887вҶ’  // footer + accent line from being clipped at the cell bottom edge
   888вҶ’   888вҶ’  const FOOTER_MIN = 800;
   889вҶ’   889вҶ’  const rawTop = Math.floor(safeRemaining * 0.45);
   890вҶ’   890вҶ’  const rawBottom = Math.floor(safeRemaining * 0.45);
   891вҶ’   891вҶ’  const bottomSpacing = Math.max(rawBottom, FOOTER_MIN);
   892вҶ’   892вҶ’  const topSpacing = Math.max(rawTop - Math.max(0, FOOTER_MIN - rawBottom), 400);
   893вҶ’   893вҶ’  const midSpacing = Math.max(safeRemaining - topSpacing - bottomSpacing, 0);
   894вҶ’   894вҶ’
   895вҶ’   895вҶ’  return { topSpacing, midSpacing, bottomSpacing };
   896вҶ’   896вҶ’}
   897вҶ’   897вҶ’```

   926вҶ’   926вҶ’### вҡ пёҸ CRITICAL вҖ” Cover Section Non-Negotiables (ALL Recipes)
   927вҶ’   927вҶ’
   928вҶ’   928вҶ’These 3 properties are MANDATORY for every cover implementation (R1вҖ“R7). Omitting ANY of them causes cover layout failure:
   929вҶ’   929вҶ’
   930вҶ’   930вҶ’1. **Cover section margin = 0**: The cover MUST be in its own section with `page.margin: { top: 0, bottom: 0, left: 0, right: 0 }`. Non-zero margins shrink the wrapper away from page edges вҶ’ white gaps around the cover ("cover not filling the page"). This is the #1 cause of broken cover layouts.
   931вҶ’   931вҶ’
   932вҶ’   932вҶ’2. **Wrapper row exact height**: The outer wrapper table row MUST set `height: { value: 16838, rule: "exact" }`. Without this, content overflow pushes to page 2, or insufficient content leaves bottom whitespace.
   933вҶ’   933вҶ’
   934вҶ’   934вҶ’3. **Wrapper table borders = allNoBorders**: MUST explicitly set `borders: allNoBorders`. Default docx-js borders add ~8 twips per edge. MS Office includes border thickness in exact-height calculation вҶ’ total exceeds 16838 вҶ’ blank page 2 (WPS is lenient, MS Office is strict).
   935вҶ’   935вҶ’
   936вҶ’   936вҶ’**Cover section template (copy this for every Recipe):**
   937вҶ’   937вҶ’```js
   938вҶ’   938вҶ’sections: [

   943вҶ’   943вҶ’        size: { width: 11906, height: 16838 },

  1445вҶ’  1445вҶ’### Recipe R5: Clean White (Academic)
  1446вҶ’  1446вҶ’
  1447вҶ’  1447вҶ’**Visual:** Pure white background + school name + centered title + 2-column meta info table with underlines + footer
  1448вҶ’  1448вҶ’**Use case:** academic thesis, standards documents
  1449вҶ’  1449вҶ’**Architecture:** 16838 outer wrapper (white fill, invisible) + cell margins for page margin simulation. No top/bottom decorative lines.
  1450вҶ’  1450вҶ’
  1451вҶ’  1451вҶ’**Meta info table rules (cross-engine safe):**
  1452вҶ’  1452вҶ’- 2-column table: label + value, **percentage widths only** (`WidthType.PERCENTAGE`)
  1453вҶ’  1453вҶ’- **Table width is adaptive:** 55вҖ“75% of page, calculated by `calcR5MetaLayout()`. Table centered via `alignment: CENTER`.
  1454вҶ’  1454вҶ’- **Label column:** adaptive 25вҖ“45% of table, **LEFT aligned**, plain text with "пјҡ" appended. NO full-width space padding, NO right-alignment.
  1455вҶ’  1455вҶ’- **Label column borders:** none (no bottom border on label cells).
  1456вҶ’  1456вҶ’- **Value column:** remaining %, **LEFT aligned**, `bottom: single sz=4` border = fixed-length underline (consistent length for all rows regardless of text).
  1457вҶ’  1457вҶ’- No left/right/top borders on either column.
  1458вҶ’  1458вҶ’- вҡ пёҸ Do NOT use DXA widths, full-width space padding (`\u3000`), spacer columns, or tab stops вҖ” WPS renders them inconsistently.
  1459вҶ’  1459вҶ’- вҡ пёҸ Do NOT use `margins.top` on the wrapper cell вҖ” use `spacing.before` on first paragraph instead.
  1460вҶ’  1460вҶ’
  1461вҶ’  1461вҶ’**Known limitation:** When meta lines вүҘ 6 AND title has 3 lines, MS Office may render content slightly taller than WPS, potentially clipping the footer line. Mitigate by reducing `midSpacing` or using a smaller title font.

  1722вҶ’  1722вҶ’1. **Anti-overlap**: If >6 x-axis labels, rotate 45В° (`plt.xticks(rotation=45, ha='right')`)
  1723вҶ’  1723вҶ’2. **Anti-stretch**: Always set figure size explicitly (`fig, ax = plt.subplots(figsize=(10, 6))`)
  1724вҶ’  1724вҶ’3. **Aspect ratio (CRITICAL)**: When embedding in docx, MUST read actual image dimensions and calculate height proportionally. NEVER hardcode both width and height вҖ” pie charts become ellipses, radar charts become diamonds.
  1725вҶ’  1725вҶ’   ```js
  1726вҶ’  1726вҶ’   const sizeOf = require("image-size");
  1727вҶ’  1727вҶ’   const dims = sizeOf(chartBuffer);
  1728вҶ’  1728вҶ’   const displayWidth = 500;
  1729вҶ’  1729вҶ’   const displayHeight = Math.round(displayWidth * (dims.height / dims.width));
  1730вҶ’  1730вҶ’   // transformation: { width: displayWidth, height: displayHeight }
  1731вҶ’  1731вҶ’   ```
  1732вҶ’  1732вҶ’4. **DPI**: Save at 200+ DPI (`plt.savefig("chart.png", dpi=200, bbox_inches="tight")`)
  1733вҶ’  1733вҶ’5. **Colors**: Use palette accent color for primary data series
  1734вҶ’  1734вҶ’6. **Legend**: Place outside plot area if >4 series
  1735вҶ’  1735вҶ’7. **Square charts**: Pie and radar charts MUST use `figsize=(8, 8)` (equal width/height) to preserve circular/radial shape
  1736вҶ’  1736вҶ’7. **Grid**: Light gray grid (`ax.grid(True, alpha=0.3)`)
